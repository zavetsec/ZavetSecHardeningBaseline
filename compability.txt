OVERVIEW
================================================================================

ZavetSecHardeningBaseline is a production-grade PowerShell hardening script built
for information security professionals working with Windows environments. Its
primary value lies in combining audit, enforcement, and rollback into a single
scriptable, PsExec-compatible tool that produces machine-readable backups and
human-readable HTML reports.

TARGET AUDIENCE
  - SOC analysts performing baseline compliance checks
  - DFIR engineers establishing a known-good state post-incident
  - System administrators deploying hardening at scale
  - Security engineers building repeatable, auditable hardening pipelines

DESIGN PRINCIPLES
  - No external dependencies. No agents. No internet required.
  - PowerShell 5.1 only — works on every supported Windows version.
  - Idempotent: run Apply twice, the result is the same.
  - Safe: JSON backup created before every change; full Rollback supported.
  - Silent-mode capable: -NonInteractive flag for PsExec and automation.

COMPLIANCE MAPPING
  - CIS Microsoft Windows Benchmark (Level 1 and Level 2)
  - DISA STIG for Windows 10/Server
  - Microsoft Security Baseline
  - MITRE ATT&CK (technique references in each check)

SECTION 1 — NETWORK HARDENING (NET-001 to NET-010)
================================================================================

NET-001 | Disable LLMNR
  Severity   : HIGH
  MITRE      : T1557.001 (Adversary-in-the-Middle: LLMNR/NBT-NS Poisoning)
  CIS        : L1 18.5.4.2
  Risk       : LLMNR (Link-Local Multicast Name Resolution) broadcasts name
               queries on the local subnet when DNS fails. Tools like Responder
               intercept these broadcasts and respond with attacker-controlled
               addresses, capturing NetNTLM hashes for offline cracking or relay
               attacks. This is one of the most common initial access techniques
               in internal penetration tests.
  Remediation: Sets HKLM\SOFTWARE\Policies\Microsoft\Windows NT\DNSClient\
               EnableMulticast = 0 via registry and Group Policy path.
  After fix  : LLMNR queries are silently dropped. No user-visible impact in
               environments with properly configured DNS.

NET-002 | Disable mDNS
  Severity   : MEDIUM
  MITRE      : T1557 (Adversary-in-the-Middle)
  CIS        : L2
  Risk       : Multicast DNS (mDNS) operates similarly to LLMNR on port 5353/UDP.
               It can be exploited for name poisoning on the local segment,
               particularly against modern Windows 10/11 workstations which
               have mDNS enabled by default.
  Remediation: Sets EnableMDNS = 0 in the DNSClient policy key.
  After fix  : mDNS resolution disabled. Rarely needed in corporate environments
               with centralized DNS infrastructure.

NET-003 | Disable WPAD
  Severity   : HIGH
  MITRE      : T1557 (Adversary-in-the-Middle)
  Risk       : Web Proxy Auto-Discovery (WPAD) causes Windows to search for a
               proxy configuration file. An attacker on the local network can
               serve a malicious WPAD response, routing all HTTP traffic through
               an attacker-controlled proxy and capturing credentials, session
               tokens, and sensitive data in cleartext or via SSL stripping.
  Remediation: Sets DisableWpad = 1 in WinHttp settings. Also stops the
               WinHttpAutoProxySvc service.
  After fix  : WPAD auto-detection disabled. Configured proxy settings continue
               to function normally.

NET-004 | Disable SMBv1
  Severity   : CRITICAL
  MITRE      : T1210 (Exploitation of Remote Services)
  CIS        : L1 | MS KB2696547
  Risk       : SMBv1 is the protocol exploited by EternalBlue (MS17-010), which
               was used in the WannaCry ransomware outbreak (2017) and NotPetya.
               The protocol lacks encryption, integrity checking, and modern
               authentication. It should not exist on any system in any network.
  Remediation: Disables SMB1 on both server (LanmanServer) and client (mrxsmb10
               driver Start=4) sides. Also runs Disable-WindowsOptionalFeature.
               Requires reboot for full effect.
  After fix  : SMBv1 connections rejected. Systems running only SMBv2/SMBv3 are
               unaffected. Legacy devices that only speak SMBv1 will lose access.

NET-005 | Require SMB Signing (Server)
  Severity   : HIGH
  MITRE      : T1557.001
  CIS        : L1
  Risk       : Without mandatory SMB signing, an attacker performing NTLM relay
               can forward captured authentication to an SMB server and
               authenticate as the victim. This is the basis of attacks like
               PetitPotam, PrinterBug, and classic NTLM relay chains.
  Remediation: Sets RequireSecuritySignature = 1 and EnableSecuritySignature = 1
               on LanmanServer. Also applies via Set-SmbServerConfiguration.
  After fix  : All SMB connections to this server must be signed. Clients that
               do not support signing will be rejected.

NET-006 | Require SMB Signing (Client)
  Severity   : HIGH
  MITRE      : T1557.001
  CIS        : L1
  Risk       : Client-side SMB signing ensures this machine does not connect to
               unsigned servers, preventing it from being tricked into
               authenticating to a rogue SMB server set up by an attacker.
  Remediation: Sets RequireSecuritySignature = 1 on LanmanWorkstation.
  After fix  : This machine will refuse to connect to SMB servers that do not
               offer signing. Negligible impact in modern environments.

NET-007 | Disable NetBIOS over TCP/IP (NBT-NS)
  Severity   : HIGH
  MITRE      : T1557.001
  Risk       : NetBIOS Name Service (NBT-NS) is a legacy broadcast name
               resolution protocol that has no place in modern networks. Like
               LLMNR, it is trivially poisoned by Responder and similar tools,
               enabling MITM and credential capture on the local subnet.
  Remediation: Sets NetbiosOptions = 2 on all network adapter interfaces
               (HKLM\SYSTEM\...\NetBT\Parameters\Interfaces).
  After fix  : NetBIOS name broadcasts disabled on all adapters. Name resolution
               falls back to DNS only.

NET-008 | Disable LMHOSTS Lookup
  Severity   : LOW
  CIS        : L2
  Risk       : LMHOSTS is a legacy static name resolution file predating DNS.
               Its use represents unnecessary attack surface and legacy protocol
               exposure. In rare cases it could be abused if an attacker
               gains write access to the file.
  Remediation: Sets EnableLMHOSTS = 0 in NetBT Parameters.
  After fix  : LMHOSTS file ignored. No operational impact in modern environments.

NET-009 | Restrict Anonymous Enumeration
  Severity   : HIGH
  MITRE      : T1087 (Account Discovery)
  CIS        : L1 2.3.10.2
  Risk       : By default, unauthenticated users can enumerate SAM accounts,
               local groups, and network shares via null session. This provides
               attackers with usernames for password spraying, brute force, and
               targeted attacks without any credentials.
  Remediation: Sets RestrictAnonymousSAM = 1, RestrictAnonymous = 1, and
               EveryoneIncludesAnonymous = 0 in the LSA registry key.
  After fix  : Anonymous enumeration of SAM and share information blocked.
               Authenticated access unaffected.

NET-010 | Disable Remote Registry
  Severity   : HIGH
  CIS        : L1
  Risk       : The Remote Registry service allows authenticated users to read and
               modify registry keys over the network. This is a significant
               lateral movement enabler — attackers with valid credentials can
               remotely alter security settings, persistence mechanisms, and
               service configurations.
  Remediation: Stops and disables the RemoteRegistry service.
  After fix  : Remote registry access completely blocked. Tools that rely on
               remote registry (some monitoring agents) may need reconfiguration.

SECTION 2 — CREDENTIAL PROTECTION (CRED-001 to CRED-006)
================================================================================

CRED-001 | Disable WDigest
  Severity   : CRITICAL
  MITRE      : T1003.001 (OS Credential Dumping: LSASS Memory)
  Risk       : The WDigest authentication provider caches credentials in
               plaintext in LSASS memory. Tools like Mimikatz can extract these
               cleartext passwords from memory with a single command. This
               remains one of the most impactful post-exploitation techniques
               when WDigest is enabled (as it is by default on Windows versions
               prior to 8.1/2012R2 and can be re-enabled by attackers).
  Remediation: Sets UseLogonCredential = 0 in the WDigest provider key.
  After fix  : Cleartext credentials no longer cached in LSASS. Mimikatz
               sekurlsa::wdigest will return empty results.

CRED-002 | Enable LSA Protection (RunAsPPL)
  Severity   : CRITICAL
  MITRE      : T1003.001
  CIS        : MS KB3033929
  Risk       : Without Protected Process Light, any process running as
               Administrator or SYSTEM can inject into LSASS and dump
               credentials. LSA PPL marks the lsass.exe process as a protected
               process, requiring signed kernel-mode code to interact with it.
               This significantly raises the bar for credential theft attacks.
  Remediation: Sets RunAsPPL = 1 in HKLM\SYSTEM\...\Control\Lsa. Requires
               reboot to take effect.
  After fix  : LSASS protected as PPL. Standard Mimikatz credential dump
               techniques fail without a kernel-level driver (e.g., mimidrv.sys).

CRED-003 | Enable Credential Guard
  Severity   : HIGH
  MITRE      : T1003
  Risk       : Even with LSA PPL, NTLM hashes and Kerberos tickets stored in
               LSASS memory can be extracted by sufficiently privileged attackers.
               Credential Guard moves these secrets into a Virtualization-Based
               Security (VBS) isolated environment inaccessible to the main OS,
               even at SYSTEM privilege.
  Remediation: Enables VBS and Credential Guard via DeviceGuard policy keys and
               LsaCfgFlags = 1. Requires UEFI, Secure Boot, and VBS-capable
               hardware. Requires reboot.
  After fix  : NTLM hashes and Kerberos TGTs stored in VBS-isolated memory.
               Pass-the-Hash and Pass-the-Ticket attacks severely impaired.

CRED-004 | NTLMv2 Only (LmCompatibilityLevel = 5)
  Severity   : HIGH
  MITRE      : T1110 (Brute Force)
  CIS        : L1 2.3.11.7
  Risk       : NTLMv1 and LM authentication responses use weak cryptography
               (DES-based) that can be cracked in seconds with modern hardware
               (e.g., using rainbow tables or GPU cracking). LmCompatibilityLevel
               controls which challenge-response protocol is used.
  Remediation: Sets LmCompatibilityLevel = 5 (send NTLMv2 only, refuse LM
               and NTLMv1 responses).
  After fix  : Only NTLMv2 responses sent and accepted. Legacy systems that
               only support LM/NTLMv1 will be unable to authenticate.

CRED-005 | Disable LM Hash Storage
  Severity   : HIGH
  CIS        : L1 2.3.11.3
  Risk       : When a user sets or changes their password, Windows can store a
               LAN Manager hash alongside the NTLM hash in the SAM database.
               LM hashes split the password into two 7-character chunks and use
               DES encryption — trivially crackable with any GPU cracking tool.
  Remediation: Sets NoLMHash = 1 in the LSA key. Password changes after this
               setting will no longer generate LM hashes.
  After fix  : No new LM hashes stored. Existing LM hashes in SAM remain until
               users change their passwords. Note: LM hashes already stored are
               not removed by this setting.

CRED-006 | Require 128-bit NTLM Session Security
  Severity   : MEDIUM
  CIS        : L1 2.3.11.9
  Risk       : NTLM session security flags control the minimum encryption
               strength for NTLM-authenticated sessions. Without enforcing
               NTLMv2 session security and 128-bit encryption, NTLM sessions
               may negotiate weaker encryption susceptible to interception and
               decryption.
  Remediation: Sets NTLMMinServerSec and NTLMMinClientSec = 537395200
               (0x20000000 | 0x80000000 = NTLMv2 + 128-bit).
  After fix  : All NTLM-authenticated sessions require NTLMv2 session security
               with 128-bit encryption. Older clients that cannot negotiate
               this level will be refused.

SECTION 3 — POWERSHELL HARDENING (PS-001 to PS-005)
================================================================================

PS-001 | Script Block Logging (Event 4104)
  Severity   : HIGH
  MITRE      : T1059.001 (Command and Scripting Interpreter: PowerShell)
  CIS        : L1
  Risk       : PowerShell is the most-used attacker tool on Windows. Without
               Script Block Logging, all PowerShell activity — including encoded
               commands, obfuscated loaders, and in-memory execution — is
               invisible to defenders. Attackers routinely use base64 encoding,
               string concatenation, and variable substitution to evade
               string-match detection.
  Remediation: Enables EnableScriptBlockLogging and
               EnableScriptBlockInvocationLogging in the PowerShell policy key.
               Events written to Microsoft-Windows-PowerShell/Operational log.
  After fix  : All executed PowerShell code logged as Event 4104, including
               decoded content. Obfuscation is defeated at the execution layer.
               Forward to SIEM for correlation.

PS-002 | Module Logging (Event 4103)
  Severity   : HIGH
  MITRE      : T1059.001
  CIS        : L1
  Risk       : Module logging records the full pipeline execution output for
               all PowerShell modules. Combined with Script Block Logging, it
               provides a complete picture of what PowerShell commands were
               executed and what they produced — essential for DFIR timelines.
  Remediation: Enables EnableModuleLogging and adds a wildcard (*) module name
               entry to log all modules.
  After fix  : Detailed pipeline execution logged as Event 4103. High-volume
               on active systems — size the log accordingly.

PS-003 | Transcription
  Severity   : MEDIUM
  MITRE      : T1059.001
  Risk       : Transcription creates a text file record of every PowerShell
               session, capturing all input and output. Unlike event log entries,
               transcripts survive log clearing attacks (they are plain files
               on disk). They are invaluable during incident response for
               reconstructing attacker activity.
  Remediation: Enables EnableTranscripting with output to
               C:\ProgramData\PSTranscripts. Includes invocation headers
               (timestamp, username, PID).
  After fix  : All interactive and non-interactive PowerShell sessions produce
               transcript files. Note: transcripts are local to the machine —
               consider centralizing them or shipping to a log aggregator.

PS-004 | Disable PowerShell v2 Engine
  Severity   : HIGH
  MITRE      : T1059.001
  Risk       : PowerShell v2 predates Script Block Logging, AMSI, Constrained
               Language Mode, and modern security controls. Attackers routinely
               use "powershell.exe -version 2" to instantiate the v2 engine and
               bypass all of the above in one step. If the v2 Windows Feature is
               installed, this bypass remains available regardless of other PS
               hardening.
  Remediation: Disables MicrosoftWindowsPowerShellV2 and
               MicrosoftWindowsPowerShellV2Root optional features. Requires
               reboot.
  After fix  : "powershell -version 2" fails with a feature unavailable error.
               All PowerShell execution occurs under the current version with
               full logging and AMSI coverage.

PS-005 | Execution Policy (RemoteSigned)
  Severity   : MEDIUM
  CIS        : L1
  Risk       : Default Execution Policy (Restricted or Bypass in some
               configurations) allows or prevents script execution. While not
               a security boundary per se (it is trivially bypassed), setting
               RemoteSigned via GPO path provides a documented baseline and
               slows unsophisticated attackers or accidental execution of
               downloaded scripts.
  Remediation: Sets ExecutionPolicy = RemoteSigned and EnableScripts = 1 in
               the machine-level PowerShell policy key.
  After fix  : Downloaded/remote scripts must be signed. Local scripts run
               without restriction. Can be overridden by -ExecutionPolicy
               parameter — use AppLocker or WDAC for enforcement.

SECTION 4 — AUDIT POLICY (AUD-001 to AUD-027)
================================================================================

All audit checks use auditpol.exe with subcategory GUIDs for locale-independent
operation (works on Russian, English, and any other Windows localization).

AUD-001 | Process Creation — Success, Failure
  Events     : 4688
  Severity   : HIGH
  Risk       : Without process creation auditing, there is no event log record
               of what processes ran on a system. Attackers launching tools,
               executing payloads, or running LOLBins leave no trace.
  Value      : 4688 events with command line logging enabled form the foundation
               of process execution forensics. Required for detecting living-off-
               the-land techniques (MITRE T1218, T1059).

AUD-002 | Process Termination — Success
  Events     : 4689
  Severity   : LOW
  Value      : Useful for measuring tool execution duration and correlating
               process start/stop pairs during DFIR timeline reconstruction.

AUD-003 | Logon — Success, Failure
  Events     : 4624, 4625
  Severity   : CRITICAL
  Risk       : Logon events are the most fundamental security telemetry.
               4625 (failed logon) is essential for detecting brute force,
               password spraying, and credential stuffing. 4624 (successful
               logon) provides authentication timeline for DFIR.
  Value      : Monitor for: high-frequency 4625 (brute force), 4624 LogonType 3
               from unexpected sources (lateral movement), 4624 LogonType 10
               (remote interactive / RDP).

AUD-004 | Logoff — Success
  Events     : 4634, 4647
  Severity   : LOW
  Value      : Session duration calculation. Pairs with 4624 to establish
               session start/end in DFIR timelines.

AUD-005 | Account Lockout — Failure
  Events     : 4740
  Severity   : HIGH
  Risk       : Account lockouts indicate authentication attacks (brute force,
               password spray) or misconfigured service accounts.
  Value      : 4740 includes the source machine name — essential for locating
               the origin of lockout-causing authentication attempts.

AUD-006 | Special Logon — Success
  Events     : 4672
  Severity   : HIGH
  Risk       : 4672 is generated when an account with sensitive privileges
               (SeDebugPrivilege, SeTcbPrivilege, etc.) logs on. This covers
               administrator logons and service accounts with elevated rights.
  Value      : Baseline administrative activity. Unexpected 4672 events
               (especially for non-admin accounts) indicate privilege escalation.

AUD-007 | Kerberos Authentication Service — Success, Failure
  Events     : 4768, 4771
  Severity   : HIGH
  Value      : 4768 (TGT request) and 4771 (pre-auth failure) are required for
               detecting Kerberoasting reconnaissance, AS-REP Roasting, and
               brute force against Kerberos. DC-level telemetry.

AUD-008 | Kerberos Service Ticket Operations — Success, Failure
  Events     : 4769
  Severity   : HIGH
  Value      : 4769 (TGS request) with RC4 encryption type (0x17) is the
               canonical Kerberoasting indicator. Essential on Domain Controllers.

AUD-009 | Credential Validation — Success, Failure
  Events     : 4776
  Severity   : HIGH
  Value      : NTLM authentication attempts against the local SAM. High failure
               rate indicates brute force or lateral movement using invalid
               credentials.

AUD-010 | User Account Management — Success, Failure
  Events     : 4720–4726, 4738, 4740
  Severity   : HIGH
  Risk       : Account creation, deletion, password changes, and enable/disable
               events are critical for detecting persistence (new accounts),
               credential manipulation, and administrative abuse.
  Value      : Any unexpected account creation or privilege modification should
               trigger an alert. Baseline and alert on deviations.

AUD-011 | Security Group Management — Success
  Events     : 4728, 4732, 4756
  Severity   : HIGH
  Value      : Changes to privileged groups (Domain Admins, Administrators,
               Remote Desktop Users). Adding an account to a privileged group
               is a primary persistence and privilege escalation technique.

AUD-012 | Audit Policy Change — Success, Failure
  Events     : 4719
  Severity   : CRITICAL
  Risk       : If an attacker can modify audit policy, they can disable the
               very logging used to detect them. 4719 is the only event that
               records such changes.
  Value      : 4719 should trigger an immediate high-priority alert. Legitimate
               audit policy changes are rare and should be change-managed.

AUD-013 | Authentication Policy Change — Success
  Events     : 4706, 4713, 4716
  Severity   : HIGH
  Value      : Changes to domain trust relationships and Kerberos policy.
               Unexpected trust additions are a common persistence technique
               (Golden Ticket infrastructure, domain backdoors).

AUD-014 | System Integrity — Success, Failure
  Events     : 4612, 4615, 4618
  Severity   : CRITICAL
  Value      : Security subsystem integrity violations. Rare in normal
               operation — any occurrence warrants immediate investigation.

AUD-015 | Security System Extension — Success
  Events     : 4610, 4614
  Severity   : HIGH
  Value      : Authentication package and security provider loading. Attackers
               can register malicious SSPs (Security Support Providers) that
               load into LSASS. This event records such loading.

AUD-016 | File System — Success, Failure
  Events     : 4663
  Severity   : MEDIUM
  Risk       : File system auditing requires SACL configuration on specific
               objects. This setting enables the audit channel; SACLs determine
               which objects generate events.
  Value      : Essential for auditing sensitive directories and files. Must be
               combined with SACL deployment for meaningful coverage.

AUD-017 | Registry — Success, Failure
  Events     : 4657
  Severity   : MEDIUM
  Value      : Registry modification auditing (requires SACLs). Useful for
               monitoring persistence-related keys (Run, Services, AppInit_DLLs)
               and security configuration keys.

AUD-018 | Sensitive Privilege Use — Success, Failure
  Events     : 4673, 4674
  Severity   : HIGH
  Value      : Use of sensitive privileges (SeDebugPrivilege, SeTakeOwnership,
               SeLoadDriver, etc.). SeLoadDriver use is a common kernel driver
               loading technique for bypassing LSA PPL.

AUD-019 | Other Object Access Events — Success, Failure
  Events     : 4698, 4702
  Severity   : MEDIUM
  Value      : Scheduled task creation and modification — a primary persistence
               mechanism (MITRE T1053.005). 4698 (task created) should be
               monitored for unexpected entries.

AUD-020 | Removable Storage — Success, Failure
  Events     : 4656, 4663
  Severity   : MEDIUM
  Value      : USB and removable media access. Essential for data exfiltration
               detection and insider threat monitoring. Generates 4663 events
               for access to removable storage objects.

AUD-021 | DPAPI Activity — Success, Failure
  Events     : 4692, 4693
  Severity   : MEDIUM
  Value      : Data Protection API operations — used by browsers, credential
               managers, and applications to encrypt stored secrets. DPAPI
               abuse (MITRE T1555.003) generates these events. Useful for
               detecting credential harvesting from browser stores.

AUD-022 | RPC Events — Success, Failure
  Events     : 5712
  Severity   : LOW
  Value      : Remote Procedure Call events. Baseline noise, but useful for
               detecting specific RPC-based lateral movement techniques like
               PetitPotam (EfsRpcOpenFileRaw) when combined with network data.

AUD-023 | Security State Change — Success
  Events     : 4608, 4609
  Severity   : HIGH
  Value      : Windows Security subsystem startup and shutdown. 4609 (shutdown)
               during expected uptime indicates system tampering or unexpected
               reboot. Essential for establishing system availability timeline.

AUD-024 | Other System Events — Success, Failure
  Events     : 5024, 5025, 5027, 5028
  Severity   : MEDIUM
  Value      : Windows Firewall service start/stop events. Firewall service
               stoppage is a common attacker action to enable lateral movement.

AUD-025 | Filtering Platform Connection — Failure
  Events     : 5157
  Severity   : LOW
  Value      : Blocked network connections. Useful for detecting lateral
               movement attempts where the firewall blocks the connection.
               Failure-only to reduce volume.

AUD-026 | Detailed File Share — Failure
  Events     : 5145
  Severity   : MEDIUM
  Value      : Detailed share access events including requested permissions.
               5145 Failure events indicate unauthorized share access attempts —
               reconnaissance and lateral movement telemetry.

AUD-027 | Network Policy Server — Success, Failure
  Events     : 6272, 6273
  Severity   : LOW
  Value      : NPS (RADIUS) authentication events. Relevant where NPS is used
               for VPN, 802.1x, or Wi-Fi authentication. 6273 (access denied)
               indicates failed network authentication attempts.

SECTION 5 — SYSTEM HARDENING (SYS-001 to SYS-010)
================================================================================

SYS-001 | Enable UAC (Full Enforcement)
  Severity   : CRITICAL
  MITRE      : T1548.002 (Abuse Elevation Control Mechanism)
  CIS        : L1
  Risk       : User Account Control prevents unauthorized privilege elevation.
               With UAC disabled or in auto-approve mode, any process running
               as a standard user can silently obtain SYSTEM-level privileges
               through auto-elevation or UAC bypass techniques without any
               user interaction.
  Remediation: Sets ConsentPromptBehaviorAdmin = 2 (consent on secure desktop),
               EnableLUA = 1, PromptOnSecureDesktop = 1.
  After fix  : Administrative actions require explicit confirmation on the
               secure desktop. UAC bypass techniques that rely on auto-elevation
               are neutralized.

SYS-002 | Disable AutoRun/AutoPlay
  Severity   : HIGH
  MITRE      : T1091 (Replication Through Removable Media)
  CIS        : L1 18.9.8
  Risk       : AutoRun automatically executes code from removable media when
               inserted. This is the primary delivery vector for USB-based
               attacks (rubber ducky, malicious USB drives). AutoPlay opens
               the media browser and can trigger execution through handler
               exploitation.
  Remediation: Sets NoDriveTypeAutoRun = 255 (disable for all drive types) and
               NoAutoplayfornonVolume = 1. Applied to both HKLM and relevant
               explorer policy keys.
  After fix  : No code executes automatically from USB drives or other
               removable media. Users must manually navigate to and execute
               files.

SYS-003 | Enable Windows Firewall (All Profiles)
  Severity   : CRITICAL
  MITRE      : T1562.004 (Impair Defenses: Disable or Modify System Firewall)
  CIS        : L1 9.1.1
  Risk       : Windows Firewall disabled on any profile exposes all listening
               services to the network. Attackers routinely disable the firewall
               post-compromise to enable inbound connections and lateral
               movement tools.
  Remediation: Enables firewall on Domain, Private, and Public profiles via
               Set-NetFirewallProfile and netsh as a fallback.
  After fix  : All profiles actively filtering. Inbound connections governed
               by configured rules. Verify existing rules do not leave
               unintended ports open.

SYS-004 | Require RDP NLA
  Severity   : HIGH
  MITRE      : T1021.001 (Remote Services: Remote Desktop Protocol)
  CIS        : L1
  Risk       : Without Network Level Authentication, RDP allows unauthenticated
               access to the logon screen. This exposes the system to pre-
               authentication vulnerabilities (e.g., BlueKeep, CVE-2019-0708)
               and enables credential brute force against the graphical logon
               screen without any authentication gate.
  Remediation: Sets UserAuthenticationRequired = 1 on the RDP-Tcp listener.
  After fix  : RDP connections require successful authentication before a
               session is created. Pre-auth exploits blocked. Brute force
               attempts must go through NLA challenge-response.

SYS-005 | Enable DEP (AlwaysOn)
  Severity   : HIGH
  CIS        : L1
  Risk       : Data Execution Prevention marks memory regions as non-executable,
               preventing shellcode injected into data sections (heap, stack)
               from executing. Without DEP, classic buffer overflow and heap
               spray attacks can execute arbitrary code.
  Remediation: Sets bcdedit nx AlwaysOn for the current boot entry. Requires
               reboot.
  After fix  : All processes covered by DEP. Shellcode in non-executable memory
               regions triggers an access violation instead of executing.
               64-bit processes already have hardware DEP; this extends coverage
               to 32-bit processes.

SYS-006 | Event Log Sizing + Retention Policy
  Severity   : HIGH
  CIS        : L1
  Risk       : Default Windows event log sizes are extremely small (20 MB for
               Security). On active systems, the Security log fills within hours
               and begins overwriting events. If retention is set to "archive"
               (AutoBackupLogFiles), Windows creates Archive-Security-*.evtx
               files indefinitely, filling system drives silently. Either
               outcome (lost events or full disk) impairs incident response.
  Remediation: Sets Security log to 1 GB, System/Application to 256 MB.
               Retention set to overwrite (Retention = 0, AutoBackupLogFiles = 0,
               wevtutil /rt:false /ab:false). Oldest events overwritten when
               full — SIEM ingestion should capture events before overwrite.
  After fix  : Extended log retention window. No archive files created on disk.
               Security log provides approximately 2-4 weeks of event history
               on typical workloads before overwrite begins.

SYS-007 | Enable DNS over HTTPS (DoH)
  Severity   : MEDIUM
  MITRE      : T1071.004 (Application Layer Protocol: DNS)
  Risk       : Plaintext DNS is trivially intercepted and manipulated on the
               local network and by ISPs. DNS-based C2 and DNS hijacking attacks
               are rendered significantly more difficult when DNS traffic is
               encrypted and authenticated via HTTPS.
  Remediation: Sets DoHPolicy = 2 (Allow DoH) in the DNSClient policy key.
               Value 3 would require DoH (more secure but may break some
               environments).
  After fix  : Windows 11 and Server 2022 will use DoH when configured resolvers
               support it. Older OS versions may not fully enforce this policy.

SYS-008 | Disable Print Spooler (Optional)
  Severity   : HIGH
  MITRE      : T1547.010
  CVE        : CVE-2021-34527 (PrintNightmare)
  Risk       : The Print Spooler service (spoolsv.exe) has a long history of
               critical vulnerabilities allowing SYSTEM-level code execution.
               PrintNightmare allows any authenticated domain user to achieve
               SYSTEM on a machine with Spooler running. If printing is not
               needed, the attack surface should be eliminated entirely.
  Remediation: Stops and disables the Spooler service. Opt-in via
               -EnablePrintSpoolerDisable flag.
  After fix  : Print Spooler attack surface completely removed. Local and
               network printing will not function. Apply only to servers and
               workstations that do not require printing.

SYS-009 | Disable DNS Client Multicast (Supplementary)
  Severity   : MEDIUM
  CIS        : L1
  Risk       : Secondary check that reinforces NET-001. Ensures multicast
               name resolution is disabled via the DNS Client service
               configuration in addition to the Group Policy path.
  After fix  : Double enforcement of LLMNR/multicast disable at both the
               policy and service configuration level.

SYS-010 | RDP Encryption Level (High)
  Severity   : HIGH
  CIS        : L1
  Risk       : Low or medium RDP encryption levels allow session traffic to
               be decrypted if captured on the network. High encryption (level 3)
               uses the highest key strength supported by the client.
  Remediation: Sets MinEncryptionLevel = 3 on the RDP-Tcp listener.
  After fix  : RDP sessions encrypted at high strength. Clients that only
               support low encryption will be refused connection.

РАЗДЕЛ 1 — СЕТЕВАЯ ЗАЩИТА (NET-001 — NET-010)

NET-001 | Отключение LLMNR
  Критичность: HIGH
  MITRE      : T1557.001 (Атака «человек посередине»: отравление LLMNR/NBT-NS)
  CIS        : L1 18.5.4.2
  Риск       : LLMNR (Link-Local Multicast Name Resolution) рассылает широковещательные
               запросы разрешения имён в локальной подсети при отказе DNS. Инструменты
               вроде Responder перехватывают эти запросы и отвечают от имени атакующего,
               захватывая NetNTLM-хэши для офлайн-взлома или ретрансляции.
               Это одна из наиболее частых техник первоначального доступа
               на внутренних тестах на проникновение.
  Исправление: Устанавливает EnableMulticast = 0 в ключе реестра политики DNSClient.
  После      : Запросы LLMNR молча отбрасываются. Нет видимого эффекта для
               пользователей при корректно настроенном DNS.

NET-002 | Отключение mDNS
  Критичность: MEDIUM
  MITRE      : T1557
  CIS        : L2
  Риск       : Multicast DNS работает аналогично LLMNR на порту 5353/UDP.
               Может использоваться для отравления имён в локальном сегменте,
               особенно против современных рабочих станций Windows 10/11,
               где mDNS включён по умолчанию.
  Исправление: Устанавливает EnableMDNS = 0 в ключе политики DNSClient.
  После      : Разрешение через mDNS отключено. Редко требуется в корпоративной
               среде с централизованной DNS-инфраструктурой.

NET-003 | Отключение WPAD
  Критичность: HIGH
  MITRE      : T1557
  Риск       : Web Proxy Auto-Discovery заставляет Windows искать файл
               конфигурации прокси. Злоумышленник в локальной сети может
               подменить WPAD-ответ, направив весь HTTP-трафик через
               подконтрольный прокси и перехватывая учётные данные, токены
               сессий и чувствительные данные.
  Исправление: Устанавливает DisableWpad = 1 в настройках WinHttp.
               Также останавливает службу WinHttpAutoProxySvc.
  После      : Автоопределение WPAD отключено. Явно заданные настройки
               прокси продолжают работать.

NET-004 | Отключение SMBv1
  Критичность: CRITICAL
  MITRE      : T1210 (Эксплуатация удалённых сервисов)
  CIS        : L1 | MS KB2696547
  Риск       : SMBv1 — протокол, использованный в атаке EternalBlue (MS17-010),
               которая стала основой эпидемий WannaCry и NotPetya. Протокол
               не имеет шифрования, проверки целостности и современной
               аутентификации. Его существование на любой системе недопустимо.
  Исправление: Отключает SMB1 на стороне сервера (LanmanServer) и клиента
               (драйвер mrxsmb10, Start=4). Также запускает
               Disable-WindowsOptionalFeature. Требует перезагрузки.
  После      : SMBv1-подключения отклоняются. Системы, работающие только
               по SMBv1, потеряют доступ.

NET-005 | Обязательная подпись SMB (сервер)
  Критичность: HIGH
  MITRE      : T1557.001
  CIS        : L1
  Риск       : Без обязательной подписи SMB злоумышленник, выполняющий ретрансляцию
               NTLM, может перенаправить перехваченную аутентификацию на SMB-сервер
               и войти под учётной записью жертвы. Это основа атак PetitPotam,
               PrinterBug и классических цепочек NTLM relay.
  Исправление: Устанавливает RequireSecuritySignature = 1 и
               EnableSecuritySignature = 1 в LanmanServer.
  После      : Все SMB-подключения к этому серверу должны быть подписаны.

NET-006 | Обязательная подпись SMB (клиент)
  Критичность: HIGH
  MITRE      : T1557.001
  CIS        : L1
  Риск       : Клиентская подпись SMB гарантирует, что данная машина не будет
               подключаться к беззнаковым серверам — это исключает возможность
               обмануть её, заставив аутентифицироваться на подставном SMB-сервере
               злоумышленника.
  Исправление: Устанавливает RequireSecuritySignature = 1 в LanmanWorkstation.
  После      : Машина откажется подключаться к SMB-серверам без поддержки подписи.

NET-007 | Отключение NetBIOS через TCP/IP (NBT-NS)
  Критичность: HIGH
  MITRE      : T1557.001
  Риск       : NetBIOS Name Service — устаревший широковещательный протокол
               разрешения имён. Как и LLMNR, тривиально отравляется инструментами
               вроде Responder, что обеспечивает атаку «человек посередине»
               и захват учётных данных в локальном сегменте.
  Исправление: Устанавливает NetbiosOptions = 2 на всех сетевых адаптерах.
  После      : Широковещательные запросы NetBIOS отключены. Разрешение имён
               работает только через DNS.

NET-008 | Отключение поиска LMHOSTS
  Критичность: LOW
  CIS        : L2
  Риск       : LMHOSTS — устаревший статический файл разрешения имён. Расширение
               поверхности атаки без практической пользы в современных средах.
  После      : Файл LMHOSTS игнорируется. Практического влияния нет.

NET-009 | Ограничение анонимного перечисления
  Критичность: HIGH
  MITRE      : T1087 (Разведка учётных записей)
  CIS        : L1 2.3.10.2
  Риск       : По умолчанию неаутентифицированные пользователи могут перечислять
               учётные записи SAM, локальные группы и сетевые ресурсы через
               нулевую сессию. Это предоставляет атакующему имена пользователей
               для подбора паролей и целевых атак без каких-либо учётных данных.
  Исправление: Устанавливает RestrictAnonymousSAM = 1, RestrictAnonymous = 1,
               EveryoneIncludesAnonymous = 0.
  После      : Анонимное перечисление SAM и ресурсов заблокировано.
               Аутентифицированный доступ не затронут.

NET-010 | Отключение Remote Registry
  Критичность: HIGH
  CIS        : L1
  Риск       : Служба Remote Registry позволяет аутентифицированным пользователям
               читать и изменять ключи реестра по сети. Значимый вектор
               горизонтального перемещения — злоумышленники с действующими
               учётными данными могут удалённо менять настройки безопасности
               и механизмы закрепления.
  Исправление: Останавливает и отключает службу RemoteRegistry.
  После      : Удалённый доступ к реестру полностью заблокирован.

РАЗДЕЛ 2 — ЗАЩИТА УЧЁТНЫХ ДАННЫХ (CRED-001 — CRED-006)
================================================================================

CRED-001 | Отключение WDigest
  Критичность: CRITICAL
  MITRE      : T1003.001 (Дамп учётных данных: память LSASS)
  Риск       : Провайдер аутентификации WDigest кэширует учётные данные в
               открытом виде в памяти LSASS. Инструменты вроде Mimikatz
               извлекают эти пароли одной командой. Одна из наиболее
               критичных техник постэксплуатации. На старых системах
               WDigest включён по умолчанию, на новых может быть включён
               повторно атакующим через тот же ключ реестра.
  Исправление: Устанавливает UseLogonCredential = 0 в ключе провайдера WDigest.
  После      : Учётные данные в открытом виде не кэшируются в LSASS.
               Команда Mimikatz sekurlsa::wdigest вернёт пустой результат.

CRED-002 | Включение LSA Protection (RunAsPPL)
  Критичность: CRITICAL
  MITRE      : T1003.001
  Риск       : Без Protected Process Light любой процесс с правами
               Администратора или SYSTEM может внедриться в LSASS и извлечь
               учётные данные. LSA PPL отмечает lsass.exe как защищённый
               процесс — для взаимодействия с ним требуется подписанный
               код уровня ядра. Существенно повышает планку для атак
               на учётные данные.
  Исправление: Устанавливает RunAsPPL = 1. Требует перезагрузки.
  После      : LSASS защищён как PPL. Стандартные техники дампа через
               Mimikatz не работают без драйвера уровня ядра (mimidrv.sys).

CRED-003 | Включение Credential Guard
  Критичность: HIGH
  MITRE      : T1003
  Риск       : Даже при включённом LSA PPL NTLM-хэши и билеты Kerberos в памяти
               LSASS могут быть извлечены достаточно привилегированными
               атакующими. Credential Guard перемещает эти секреты в изолированную
               среду VBS, недоступную из основной ОС даже с правами SYSTEM.
  Исправление: Включает VBS и Credential Guard через ключи политики DeviceGuard
               и LsaCfgFlags = 1. Требует UEFI, Secure Boot и поддержки VBS.
               Требует перезагрузки.
  После      : NTLM-хэши и TGT Kerberos в изолированной памяти VBS.
               Атаки Pass-the-Hash и Pass-the-Ticket существенно затруднены.

CRED-004 | Только NTLMv2 (LmCompatibilityLevel = 5)
  Критичность: HIGH
  MITRE      : T1110 (Подбор паролей)
  CIS        : L1 2.3.11.7
  Риск       : NTLMv1 и LM используют слабую криптографию (DES), которая
               взламывается за секунды на современном железе с использованием
               радужных таблиц или GPU. LmCompatibilityLevel управляет тем,
               какой протокол challenge-response используется.
  Исправление: Устанавливает LmCompatibilityLevel = 5 (только NTLMv2,
               отказ от LM и NTLMv1).
  После      : Используются и принимаются только NTLMv2-ответы. Устаревшие
               системы, поддерживающие только LM/NTLMv1, не смогут
               аутентифицироваться.

CRED-005 | Запрет хранения LM-хэшей
  Критичность: HIGH
  CIS        : L1 2.3.11.3
  Риск       : При установке пароля Windows может сохранять LM-хэш рядом с
               NTLM-хэшем в базе SAM. LM разбивает пароль на два блока по 7
               символов и использует DES — тривиально взламывается любым
               инструментом GPU-взлома.
  Исправление: Устанавливает NoLMHash = 1 в ключе LSA. Новые пароли после
               этого не будут генерировать LM-хэши.
  После      : Новые LM-хэши не создаются. Существующие в SAM сохраняются
               до смены пароля пользователями.

CRED-006 | Требование 128-битной защиты сессии NTLM
  Критичность: MEDIUM
  CIS        : L1 2.3.11.9
  Риск       : Флаги защиты сессии NTLM определяют минимальную стойкость
               шифрования для NTLM-аутентифицированных сессий. Без принудительного
               использования NTLMv2 и 128-битного шифрования сессии могут
               согласовывать слабое шифрование, уязвимое к перехвату.
  Исправление: Устанавливает NTLMMinServerSec и NTLMMinClientSec = 537395200
               (NTLMv2 + 128-бит).
  После      : Все NTLM-сессии используют NTLMv2 с 128-битным шифрованием.

РАЗДЕЛ 3 — ЗАЩИТА POWERSHELL (PS-001 — PS-005)
================================================================================

PS-001 | Script Block Logging (событие 4104)
  Критичность: HIGH
  MITRE      : T1059.001 (PowerShell)
  Риск       : PowerShell — наиболее используемый инструмент атакующих на
               Windows. Без Script Block Logging вся активность PowerShell —
               включая закодированные команды, обфусцированные загрузчики
               и выполнение в памяти — невидима для защитников. Атакующие
               систематически используют base64, конкатенацию строк и
               подстановку переменных для обхода сигнатурного обнаружения.
  Исправление: Включает EnableScriptBlockLogging и
               EnableScriptBlockInvocationLogging. События пишутся в
               Microsoft-Windows-PowerShell/Operational.
  После      : Весь выполняемый PS-код логируется как событие 4104, включая
               декодированное содержимое. Обфускация нейтрализована на
               уровне выполнения. Пересылать в SIEM для корреляции.

PS-002 | Module Logging (событие 4103)
  Критичность: HIGH
  MITRE      : T1059.001
  Риск       : Логирование модулей записывает полный вывод конвейера выполнения
               для всех PS-модулей. В сочетании со Script Block Logging даёт
               полную картину выполненных команд и их результатов.
  Исправление: Включает EnableModuleLogging и добавляет запись с подстановочным
               знаком (*) для логирования всех модулей.
  После      : Детальное выполнение конвейера логируется как событие 4103.
               Большой объём на активных системах — учитывать при размере лога.

PS-003 | Транскрипция
  Критичность: MEDIUM
  MITRE      : T1059.001
  Риск       : Транскрипция создаёт текстовый файл с записью каждой PS-сессии,
               сохраняя весь ввод и вывод. В отличие от записей журнала событий,
               транскрипты переживают атаки с очисткой лога (это обычные файлы
               на диске). Бесценны при реагировании на инциденты для восстановления
               действий атакующего.
  Исправление: Включает EnableTranscripting с выводом в
               C:\ProgramData\PSTranscripts, включая заголовки с временной
               меткой, именем пользователя и PID.
  После      : Все PS-сессии создают файлы транскриптов. Рекомендуется
               централизация или пересылка в агрегатор логов.

PS-004 | Отключение движка PowerShell v2
  Критичность: HIGH
  MITRE      : T1059.001
  Риск       : PowerShell v2 появился раньше Script Block Logging, AMSI,
               Constrained Language Mode и других современных защит. Атакующие
               регулярно используют "powershell.exe -version 2" для запуска
               движка v2 и обхода всего вышеперечисленного одной командой.
               Пока Feature v2 установлена — этот обход доступен независимо
               от остального харденинга PS.
  Исправление: Отключает Optional Features PowerShellV2 и PowerShellV2Root.
               Требует перезагрузки.
  После      : "powershell -version 2" завершается с ошибкой недоступности
               компонента. Всё выполнение PS — под текущей версией с полным
               покрытием логирования и AMSI.

PS-005 | Политика выполнения (RemoteSigned)
  Критичность: MEDIUM
  CIS        : L1
  Риск       : Политика выполнения по умолчанию может допускать запуск
               неподписанных загруженных скриптов. Не является границей
               безопасности (тривиально обходится), но задаёт задокументированный
               базовый уровень и замедляет неопытных атакующих.
  Исправление: Устанавливает ExecutionPolicy = RemoteSigned на уровне машины
               через ключ политики.
  После      : Загруженные/удалённые скрипты должны быть подписаны. Локальные
               скрипты запускаются без ограничений. Для принудительного
               исполнения используйте AppLocker или WDAC.

РАЗДЕЛ 4 — ПОЛИТИКА АУДИТА (AUD-001 — AUD-027)
================================================================================

Все проверки аудита используют auditpol.exe с GUID подкатегорий для
независимости от языка интерфейса (работает на русской, английской и любой
другой локализации Windows).

AUD-001 | Создание процессов — Success, Failure
  События    : 4688
  Критичность: HIGH
  Ценность   : Без аудита создания процессов нет журнальной записи о том, что
               запускалось на системе. 4688 с логированием командной строки —
               основа форензики выполнения процессов. Необходим для обнаружения
               LOLBins (T1218, T1059).

AUD-002 | Завершение процессов — Success
  События    : 4689
  Критичность: LOW
  Ценность   : Расчёт длительности работы инструментов, корреляция пар
               старт/стоп при построении таймлайна инцидента.

AUD-003 | Вход в систему — Success, Failure
  События    : 4624, 4625
  Критичность: CRITICAL
  Ценность   : Фундаментальная телеметрия безопасности. 4625 (неудачный вход) —
               обязателен для обнаружения брутфорса, распыления паролей,
               подстановки учётных данных. 4624 даёт таймлайн аутентификации.
               Мониторинг: высокочастотные 4625 (брутфорс); 4624 LogonType 3
               из неожиданных источников (горизонтальное перемещение);
               4624 LogonType 10 (RDP).

AUD-004 | Выход из системы — Success
  События    : 4634, 4647
  Критичность: LOW
  Ценность   : Расчёт длительности сессии. Пара с 4624 для таймлайна DFIR.

AUD-005 | Блокировка учётной записи — Failure
  События    : 4740
  Критичность: HIGH
  Ценность   : Блокировки указывают на атаки аутентификации или неправильно
               настроенные служебные учётки. 4740 содержит имя машины-источника.

AUD-006 | Особый вход — Success
  События    : 4672
  Критичность: HIGH
  Ценность   : Вход учётной записи с чувствительными привилегиями.
               Неожиданные 4672 для неадминистраторских учёток указывают
               на эскалацию привилегий.

AUD-007 | Служба аутентификации Kerberos — Success, Failure
  События    : 4768, 4771
  Критичность: HIGH
  Ценность   : 4768 (запрос TGT) и 4771 (сбой предаутентификации) — требуются
               для обнаружения разведки Kerberoasting, AS-REP Roasting и
               брутфорса Kerberos. Телеметрия уровня DC.

AUD-008 | Операции с билетами Kerberos — Success, Failure
  События    : 4769
  Критичность: HIGH
  Ценность   : 4769 (запрос TGS) с типом шифрования RC4 (0x17) — канонический
               индикатор Kerberoasting.

AUD-009 | Проверка учётных данных — Success, Failure
  События    : 4776
  Критичность: HIGH
  Ценность   : Попытки NTLM-аутентификации к локальному SAM. Высокая частота
               сбоев = брутфорс или горизонтальное перемещение.

AUD-010 | Управление учётными записями — Success, Failure
  События    : 4720–4726, 4738, 4740
  Критичность: HIGH
  Ценность   : Создание, удаление, изменение паролей, блокировка учёток —
               критичны для обнаружения закрепления (новые учётки) и
               манипуляций с учётными данными.

AUD-011 | Управление группами безопасности — Success
  События    : 4728, 4732, 4756
  Критичность: HIGH
  Ценность   : Изменения в привилегированных группах (Domain Admins,
               Administrators, Remote Desktop Users). Добавление в
               привилегированную группу — основная техника закрепления
               и эскалации.

AUD-012 | Изменение политики аудита — Success, Failure
  События    : 4719
  Критичность: CRITICAL
  Ценность   : Единственное событие, фиксирующее изменения в политике аудита.
               Атакующий, отключивший аудит, устраняет улики. 4719 должен
               вызывать немедленный высокоприоритетный алерт.

AUD-013 | Изменение политики аутентификации — Success
  События    : 4706, 4713, 4716
  Критичность: HIGH
  Ценность   : Изменения доверия между доменами и политики Kerberos.
               Неожиданное добавление доверия — техника закрепления
               (инфраструктура Golden Ticket).

AUD-014 | Целостность системы — Success, Failure
  События    : 4612, 4615, 4618
  Критичность: CRITICAL
  Ценность   : Нарушения целостности подсистемы безопасности. Редки в
               штатной работе — любое появление требует немедленного
               расследования.

AUD-015 | Расширение системы безопасности — Success
  События    : 4610, 4614
  Критичность: HIGH
  Ценность   : Загрузка пакетов аутентификации и провайдеров безопасности.
               Атакующие могут регистрировать вредоносные SSP, загружаемые
               в LSASS. Это событие фиксирует такую загрузку.

AUD-016 | Файловая система — Success, Failure
  События    : 4663
  Критичность: MEDIUM
  Ценность   : Аудит файловой системы требует настройки SACL на конкретных
               объектах. Этот параметр включает канал аудита; SACL определяют,
               какие объекты генерируют события.

AUD-017 | Реестр — Success, Failure
  События    : 4657
  Критичность: MEDIUM
  Ценность   : Аудит изменений реестра (требует SACL). Полезен для мониторинга
               ключей закрепления (Run, Services, AppInit_DLLs) и ключей
               конфигурации безопасности.

AUD-018 | Использование чувствительных привилегий — Success, Failure
  События    : 4673, 4674
  Критичность: HIGH
  Ценность   : Использование привилегий SeDebugPrivilege, SeTakeOwnership,
               SeLoadDriver и др. Использование SeLoadDriver — распространённая
               техника загрузки драйвера ядра для обхода LSA PPL.

AUD-019 | Другие события доступа к объектам — Success, Failure
  События    : 4698, 4702
  Критичность: MEDIUM
  Ценность   : Создание и изменение запланированных задач — основной механизм
               закрепления (MITRE T1053.005). 4698 (задача создана) следует
               мониторить на предмет неожиданных записей.

AUD-020 | Съёмные носители — Success, Failure
  События    : 4656, 4663
  Критичность: MEDIUM
  Ценность   : Доступ к USB и съёмным носителям. Необходим для обнаружения
               утечки данных и мониторинга инсайдерских угроз.

AUD-021 | Активность DPAPI — Success, Failure
  События    : 4692, 4693
  Критичность: MEDIUM
  Ценность   : Операции Data Protection API — используются браузерами,
               менеджерами паролей и приложениями для шифрования секретов.
               Злоупотребление DPAPI (MITRE T1555.003) генерирует эти события.
               Полезно для обнаружения сбора учётных данных из браузеров.

AUD-022 | События RPC — Success, Failure
  События    : 5712
  Критичность: LOW
  Ценность   : Вызовы RPC. Полезны для обнаружения конкретных техник
               горизонтального перемещения через RPC (PetitPotam) в сочетании
               с сетевыми данными.

AUD-023 | Изменение состояния безопасности — Success
  События    : 4608, 4609
  Критичность: HIGH
  Ценность   : Запуск и остановка подсистемы безопасности Windows. 4609
               (остановка) в период ожидаемой работы указывает на вмешательство
               или неожиданную перезагрузку.

AUD-024 | Другие системные события — Success, Failure
  События    : 5024, 5025, 5027, 5028
  Критичность: MEDIUM
  Ценность   : События запуска/остановки службы брандмауэра Windows.
               Остановка брандмауэра — типичное действие атакующего для
               обеспечения горизонтального перемещения.

AUD-025 | Подключения платформы фильтрации — Failure
  События    : 5157
  Критичность: LOW
  Ценность   : Заблокированные сетевые подключения. Полезно для обнаружения
               попыток горизонтального перемещения, заблокированных брандмауэром.

AUD-026 | Детальный доступ к файловым ресурсам — Failure
  События    : 5145
  Критичность: MEDIUM
  Ценность   : Детализированные события доступа к ресурсам. Сбои 5145 указывают
               на несанкционированные попытки доступа к шарам — телеметрия
               разведки и горизонтального перемещения.

AUD-027 | Сервер сетевой политики — Success, Failure
  События    : 6272, 6273
  Критичность: LOW
  Ценность   : События аутентификации NPS (RADIUS). Актуально при использовании
               NPS для VPN, 802.1x или Wi-Fi аутентификации. 6273 (отказ в
               доступе) указывает на неудачные попытки сетевой аутентификации.

РАЗДЕЛ 5 — СИСТЕМНАЯ ЗАЩИТА (SYS-001 — SYS-010)
================================================================================

SYS-001 | Включение UAC (полный контроль)
  Критичность: CRITICAL
  MITRE      : T1548.002 (Злоупотребление механизмом контроля привилегий)
  CIS        : L1
  Риск       : Без UAC или в режиме автоодобрения любой процесс стандартного
               пользователя может без взаимодействия с пользователем получить
               привилегии SYSTEM через техники обхода UAC или автоповышение.
  Исправление: Устанавливает ConsentPromptBehaviorAdmin = 2 (запрос на защищённом
               рабочем столе), EnableLUA = 1, PromptOnSecureDesktop = 1.
  После      : Административные действия требуют явного подтверждения на защищённом
               рабочем столе. Техники обхода через автоповышение нейтрализованы.

SYS-002 | Отключение AutoRun/AutoPlay
  Критичность: HIGH
  MITRE      : T1091 (Распространение через съёмные носители)
  CIS        : L1 18.9.8
  Риск       : AutoRun автоматически выполняет код со съёмных носителей при
               их подключении. Основной вектор доставки USB-атак (rubber ducky,
               вредоносные USB-накопители).
  Исправление: Устанавливает NoDriveTypeAutoRun = 255 (отключение для всех
               типов дисков) и NoAutoplayfornonVolume = 1.
  После      : Код со съёмных носителей не выполняется автоматически.
               Пользователи должны самостоятельно открывать файлы.

SYS-003 | Включение брандмауэра Windows (все профили)
  Критичность: CRITICAL
  MITRE      : T1562.004 (Нарушение средств защиты: отключение брандмауэра)
  CIS        : L1 9.1.1
  Риск       : Отключение брандмауэра на любом профиле открывает все
               прослушивающие сервисы для сети. Атакующие регулярно отключают
               брандмауэр после компрометации для входящих подключений.
  Исправление: Включает брандмауэр на профилях Домен, Частный и Публичный.
  После      : Все профили активно фильтруют трафик. Убедитесь, что
               существующие правила не оставляют непреднамеренно открытые порты.

SYS-004 | Требование NLA для RDP
  Критичность: HIGH
  MITRE      : T1021.001 (Удалённые сервисы: RDP)
  CIS        : L1
  Риск       : Без NLA RDP предоставляет неаутентифицированный доступ к экрану
               входа. Уязвимости предаутентификации (BlueKeep, CVE-2019-0708)
               и брутфорс против графического экрана входа работают без
               какого-либо шлюза аутентификации.
  Исправление: Устанавливает UserAuthenticationRequired = 1 для слушателя RDP.
  После      : RDP-подключения требуют успешной аутентификации до создания
               сессии. Предаутентификационные атаки заблокированы.

SYS-005 | Включение DEP (AlwaysOn)
  Критичность: HIGH
  CIS        : L1
  Риск       : DEP помечает области памяти как невыполнимые, предотвращая
               выполнение шеллкода, внедрённого в секции данных (куча, стек).
               Без DEP классические атаки переполнения буфера и heap spray
               могут выполнять произвольный код.
  Исправление: Устанавливает bcdedit nx AlwaysOn. Требует перезагрузки.
  После      : Все процессы охвачены DEP. Шеллкод в невыполнимых областях
               памяти вызывает нарушение доступа вместо выполнения.

SYS-006 | Размер и политика хранения журналов
  Критичность: HIGH
  CIS        : L1
  Риск       : Стандартные размеры журналов Windows крайне малы (20 МБ для
               Security). На активных системах Security-лог заполняется за
               часы и начинает перезаписывать события. При настройке
               "архивировать" (AutoBackupLogFiles) Windows бесконечно создаёт
               файлы Archive-Security-*.evtx, тихо заполняя системный диск.
               Оба исхода нарушают реагирование на инциденты.
  Исправление: Security: 1 ГБ, System/Application: 256 МБ. Retention = 0
               (перезаписывать), AutoBackupLogFiles = 0. Архивные файлы
               не создаются. SIEM должен забирать события до перезаписи.
  После      : Расширенное окно хранения событий без захламления диска.
               Security-журнал обеспечивает ~2–4 недели истории при типичной
               нагрузке до начала перезаписи.

SYS-007 | Политика DNS over HTTPS (DoH)
  Критичность: MEDIUM
  MITRE      : T1071.004 (DNS как протокол прикладного уровня)
  Риск       : Открытый DNS тривиально перехватывается и модифицируется в
               локальной сети и провайдерами. DNS-based C2 и перехват DNS
               существенно затруднены при шифровании и аутентификации
               DNS-трафика через HTTPS.
  Исправление: Устанавливает DoHPolicy = 2 (разрешить DoH).
  После      : Windows 11 и Server 2022 будут использовать DoH при поддержке
               настроенными резолверами.

SYS-008 | Отключение Print Spooler (опционально)
  Критичность: HIGH
  MITRE      : T1547.010
  CVE        : CVE-2021-34527 (PrintNightmare)
  Риск       : Служба Print Spooler (spoolsv.exe) имеет долгую историю
               критических уязвимостей с выполнением кода на уровне SYSTEM.
               PrintNightmare позволяет любому аутентифицированному доменному
               пользователю получить SYSTEM на машине с работающим Spooler.
  Исправление: Останавливает и отключает службу Spooler. Опциональный флаг:
               -EnablePrintSpoolerDisable.
  После      : Поверхность атаки Print Spooler полностью устранена. Локальная
               и сетевая печать не будет работать.

SYS-009 | Отключение многоадресной рассылки DNS Client (дополнительно)
  Критичность: MEDIUM
  CIS        : L1
  Ценность   : Дублирующая проверка, усиливающая NET-001. Обеспечивает
               отключение многоадресного разрешения имён как через путь
               групповой политики, так и через конфигурацию службы DNS Client.

SYS-010 | Уровень шифрования RDP (High)
  Критичность: HIGH
  CIS        : L1
  Риск       : Низкий или средний уровень шифрования RDP позволяет расшифровать
               трафик сессии при его перехвате в сети.
  Исправление: Устанавливает MinEncryptionLevel = 3 для слушателя RDP-Tcp.
  После      : RDP-сессии шифруются с высокой стойкостью. Клиенты, не
               поддерживающие высокий уровень шифрования, получат отказ
               в подключении.
